# DessertShop 项目全面优化方案

---

## 一、当前项目问题分析

### 1. 最大的硬伤：页面完成度严重不均

| 页面 | 状态 | 问题 |
|------|------|------|
| login | 较完整 | 设计感不错，是项目最佳页面 |
| layout | 较完整 | 侧边栏结构合理，但缺少工程化细节 |
| index (首页) | 半成品 | 数据全部硬编码，统计卡片是摆设 |
| dept | 较完整 | CRUD 功能完整，UI 有设计感 |
| emp | 功能有、UI 差 | 功能最复杂但视觉最粗糙，和 dept 页风格割裂 |
| order | 空文件 | 路由已注册但页面为空 |
| price | 空文件 | 路由已注册但页面为空 |
| report | 空文件 | 路由已注册但页面为空 |
| log | 空文件 | 路由已注册但页面为空 |

**结论：8 个页面中 4 个是空壳，项目完成度约 40%。** 这是最致命的问题——面试官或评审打开侧边栏的每个菜单，4 个页面是白屏。

### 2. UI 高级感不足的具体表现

**emp 页面是最大的视觉短板：**
- 表头用英文（Name / Gender / Avatar / Department / Position / Entry Date / Operation），和其他页面的中文风格完全割裂
- 没有 page-header 区域，直接一个 `<h1>员工管理</h1>` 顶在最上面
- 搜索栏、按钮区、表格之间没有卡片容器包裹，视觉层级扁平
- 没有空状态设计
- 对话框没有 destroy-on-close，表单数据可能残留
- 工作经历表单的布局挤压变形

**首页仪表盘的问题：**
- 统计数据全部硬编码，没有任何真实感
- "最近动态"也是写死的静态数据
- 没有图表（ECharts），作为数据统计入口却不展示任何图表
- 快捷操作区功能重复（侧边栏已有同样链接）

**全局性 UI 问题：**
- 没有 404 页面
- 没有 loading 骨架屏，数据加载时是白屏
- 没有页面级的面包屑导航
- 侧边栏退出登录用了 `<router-link>` + `@click.native`，行为冲突

### 3. 不像真实商业项目的工程缺陷

**路由守卫缺失：**
- 没有全局 `beforeEach` 导航守卫
- 每个需要鉴权的页面都在 `onMounted` 里手动检查 token
- 跳转用 `window.location.href = '/login'` 而非 `router.push`，会刷新整个页面

**状态管理缺失：**
- 用户信息散落在 localStorage + 各组件的 ref 中
- 没有 Pinia store，没有全局状态
- 登录信息在 layout、index、emp 三个地方各自读取 localStorage

**代码质量问题：**
- `request.js` 第 33 行：接口异常时用了 `ElMessage.success('接口访问异常')`，应该是 `error`
- emp 页面的 saveEmp 函数校验逻辑有 bug：`if(!empFormRef.value.validate)` 永远为 false
- API 函数命名不统一：`querryAllApi`（拼写错误，应为 query）、`queryPageApi`、`queryByIdApi`
- 没有 TypeScript，没有类型安全
- 没有环境变量配置（开发/生产 API 地址）

**缺失的工程基础设施：**
- 没有 Pinia 状态管理
- 没有路由守卫
- 没有权限控制
- 没有请求/响应的统一错误处理
- 没有环境变量
- 没有代码分割 / 懒加载
- 没有 ESLint / Prettier 配置
- 没有 404 页面
- 没有面包屑组件

## 二、完整优化方案

### 阶段一：工程化基础重构（优先级最高）

#### 1.1 安装依赖

```
pinia                    — 状态管理
echarts + vue-echarts    — 图表
nprogress                — 路由切换进度条
```

#### 1.2 新增文件结构

```
src/
├── stores/
│   └── user.js           — 用户状态（token、用户信息、登录/登出）
├── router/
│   └── index.js          — 重构：懒加载 + 导航守卫 + 404 路由
├── utils/
│   ├── request.js        — 修复 bug + 统一错误处理
│   └── auth.js           — token 存取工具函数
├── components/
│   ├── Breadcrumb.vue    — 面包屑组件
│   └── PageHeader.vue    — 统一的页面标题组件
├── views/
│   ├── 404.vue           — 404 页面
│   ├── order/index.vue   — 订单管理（新建）
│   ├── price/index.vue   — 价格管理（新建）
│   ├── report/index.vue  — 数据报表（新建，含 ECharts）
│   └── log/index.vue     — 操作日志（新建）
```

#### 1.3 路由重构

- 所有子页面改为懒加载 `() => import(...)`
- 添加全局 `beforeEach` 守卫：未登录跳转 /login，已登录访问 /login 跳转 /
- 添加 404 通配路由
- 路由 meta 中配置页面标题（用于面包屑）

#### 1.4 Pinia Store

```js
// stores/user.js
export const useUserStore = defineStore('user', {
  state: () => ({
    token: '',
    userInfo: null
  }),
  actions: {
    login(form) { ... },
    logout() { ... },
    getToken() { ... }
  }
})
```

所有组件统一从 store 读取用户信息，不再各自操作 localStorage。

#### 1.5 修复 request.js

- 修复 `ElMessage.success` → `ElMessage.error` 的 bug
- 401 处理改为调用 store.logout() + router.push
- 非 401 错误统一用 `ElMessage.error`

### 阶段二：UI 全面升级

#### 2.1 emp 页面重写（最大的视觉短板）

**现状问题：** 裸露的 Element Plus 组件，没有卡片容器，英文表头，没有设计感。

**优化方案：**
- 添加 page-header 区域（与 dept 页面风格统一）
- 搜索栏、操作按钮、表格全部用 `.table-card` 卡片容器包裹
- 表头改为中文
- 添加空状态设计
- 对话框添加 `destroy-on-close`
- 优化工作经历表单布局
- 添加 loading 状态
- 按钮样式与全局设计系统对齐

#### 2.2 首页仪表盘升级

**现状问题：** 全部硬编码数据，没有图表，没有真实感。

**优化方案：**
- 统计卡片数据改为"暂无数据"或从 API 获取（如果后端有的话）
- 添加 ECharts 图表区域：甜品销量趋势折线图 + 订单分类饼图
- "最近动态"改为带时间线样式的设计
- 快捷操作保留但优化交互（添加 hover 动画、描述文字）
- 添加"系统公告"或"待办事项"区域

#### 2.3 新建 4 个空页面

**order/index.vue — 订单管理：**
- 搜索栏（订单号、客户名、状态、日期范围）
- 状态标签筛选（待处理/制作中/已完成/已取消）
- 表格展示（订单号、客户、甜品、金额、状态、时间、操作）
- 订单详情抽屉
- 空状态设计

**price/index.vue — 价格管理：**
- 甜品分类标签页（蛋糕/面包/饮品/甜点）
- 卡片网格展示甜品（图片+名称+价格）
- 批量调价功能
- 价格历史记录

**report/index.vue — 数据报表：**
- ECharts 图表：月度营收趋势、甜品销量排行、订单状态分布、员工绩效
- 日期范围选择器
- 数据卡片汇总

**log/index.vue — 操作日志：**
- 日志列表（操作人、操作类型、详情、IP、时间）
- 筛选条件（操作人、类型、时间范围）
- 日志详情查看

#### 2.4 全局 UI 细节提升

- 添加 NProgress 路由切换进度条
- 添加面包屑组件到 layout header 中
- 侧边栏退出登录修复（不用 router-link，用 div + logout 方法）
- 添加 404 页面（带返回首页按钮）
- 登录页添加"记住我"功能
- 表格添加 loading 骨架屏
- 对话框打开时自动聚焦第一个输入框

### 阶段三：交互体验增强

#### 3.1 微交互动效
- 卡片 hover 时的 translateY + 阴影变化（已有基础，统一强化）
- 表格行 hover 高亮（已有，确认一致性）
- 按钮点击的 scale 反馈
- 页面切换的过渡动画（已有基础）
- 侧边栏折叠/展开的平滑过渡（已有）

#### 3.2 空状态设计
- 所有列表页面在无数据时展示统一的空状态组件
- 包含图标 + 文字 + 操作引导按钮

#### 3.3 骨架屏
- 首页统计卡片加载时显示骨架屏
- 表格加载时显示行骨架屏

### 阶段四：代码质量提升

#### 4.1 命名规范统一
- API 函数名统一为 queryXxx / addXxx / updateXxx / deleteXxx
- 修复 `querryAllApi` 的拼写错误
- 组件文件名统一

#### 4.2 组件抽取
- PageHeader 组件（标题 + 描述 + 操作按钮）
- Breadcrumb 组件
- EmptyState 空状态组件
- StatCard 统计卡片组件

#### 4.3 常量管理
- 职位列表、性别列表等抽离为 constants
- API 路径统一管理

## 三、优化优先级排序

| 优先级 | 任务 | 原因 |
|--------|------|------|
| P0 | 4 个空页面填充 | 白屏是最致命的体验问题 |
| P0 | emp 页面 UI 重写 | 和 dept 页面风格割裂，暴露项目粗糙 |
| P0 | 路由守卫 + 404 | 缺少基本的安全和体验保障 |
| P1 | Pinia 状态管理 | 消除散落的 localStorage 操作 |
| P1 | request.js bug 修复 | 影响用户体验的代码缺陷 |
| P1 | 首页 ECharts 图表 | 提升数据可视化和专业感 |
| P1 | 面包屑 + NProgress | 提升导航体验 |
| P2 | 骨架屏 + 空状态 | 提升加载体验 |
| P2 | 组件抽取复用 | 提升代码可维护性 |
| P2 | 微交互动效 | 提升视觉高级感 |
| P3 | TypeScript 迁移 | 长期可维护性（改动大，可选） |

## 四、预期效果

优化完成后，项目将具备：
- **8 个完整页面**，无白屏死角
- **统一的设计语言**，所有页面风格一致
- **ECharts 数据可视化**，首页和报表页有真实感
- **工程化基础**：路由守卫、状态管理、统一错误处理
- **商业化交互**：骨架屏、空状态、面包屑、进度条
- **代码质量**：组件复用、命名规范、bug 修复
